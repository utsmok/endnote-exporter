# Browser-local fixture corpus

This directory contains the deterministic fixture and golden corpus for local-web execution task **T002**.

## Delivered fixture set

The corpus includes the required minimum shapes:

- `supported-enl-data.zip` — supported archive containing `Library.enl` plus sibling `Library.Data/`
- `supported-enlp-equivalent.zip` — supported archive containing an `.enlp`-equivalent package directory
- `missing-db.zip` — expected failure where `sdb/sdb.eni` is absent
- `malformed-archive.zip` — expected failure where the `.zip` file is intentionally invalid
- `attachment-present.zip` — supported archive with attachment entries under `PDF/`
- `mixed-case-data-lookup.zip` — supported archive with a mixed-case `.Data` directory name
- `stress-large.zip` — larger synthetic fixture with 250 records for future performance validation

## Oracle and golden policy

The current Python desktop exporter is the parity oracle.

Goldens are generated by extracting each supported fixture and exporting it through `EndnoteExporter`. The stored XML is then normalized so repository artifacts remain machine-portable:

- attachment paths under `PDF/` are rewritten from absolute filesystem paths to `${PDF_ROOT}/...`
- non-attachment XML remains byte-stable output from the current exporter

Failure fixtures do **not** store XML goldens. Instead, the manifest records a failure classification and stable message fragment:

- `missing-database`
- `malformed-archive`

## Files

- `build_fixture_corpus.py` — rebuilds fixtures, goldens, and the manifest; can also validate the checked-in corpus
- `fixture-manifest.json` — generated manifest of fixture metadata, checksums, comparison modes, and failure classifications
- `fixtures/` — generated ZIP fixtures and malformed archive payload
- `golden/` — approved normalized XML outputs for supported fixtures
- `test_fixture_harness.py` — targeted regression test entrypoint for the checked-in corpus

## Usage

Rebuild and validate the corpus:

- `uv run testing/browser-local/build_fixture_corpus.py`

Validate the checked-in corpus only:

- `uv run testing/browser-local/build_fixture_corpus.py --validate-only`

Run the targeted regression test (if `pytest` is available in the environment):

- `uv run pytest testing/browser-local/test_fixture_harness.py`

## Notes

- Fixtures are synthetic and deterministic by design; they do not rely on proprietary EndNote binaries.
- The existing `testing/RefsEnschede.enlp.zip` sample remains a legacy reference asset and is not used as the T002 parity corpus because it does not match the SQLite-backed library shape needed for browser-local work.
